Architectural Overview

The application is built on a modern, component-based architecture designed for maintainability and scalability.

  • State Management: Global state is managed through three dedicated React Contexts:
    • AuthContext: Handles user session, roles, and login logic.
    • DataContext: Manages all application data (inventory, customers, bills) and handles synchronization with Google Drive.
    • UIContext: Controls global UI state like modals and fullscreen mode.
  • Component Structure: Components are logically organized by feature (e.g., `billing`, `inventory`) with a strong emphasis on reusability. Common, shared components like `Modal`, `Avatar`, and `Icons` are centralized in `components/common`.
  • Data Persistence: All application data is stored in a single JSON file within the admin's private Google Drive AppData folder, ensuring data security and ownership. The app synchronizes with this file on load and saves changes automatically.
